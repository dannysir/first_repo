floating-components

React Layout Library (NPM Package)
ReactTypeScriptViteNpm package
NPM PageLive Demo

Overview

VS Code처럼 패널을 드래그로 분할·이동·리사이즈할 수 있는 트리 기반 React 레이아웃 라이브러리입니다. 설계부터 구현·배포·버전 관리 전 과정에 AI 에이전트를 적극 활용해 개인의 아이디어를 실제 운영 중인 라이브러리로 만들었으며, Weekly Downloads 총합 1,500+를 달성했습니다.

트리 기반 레이아웃 설계

  • 중첩 분할 레이아웃을 N-ary 트리 구조로 설계했습니다.
  • 렌더링(TreeLayout)과 상태 관리(useLayoutTree)를 분리해 독립적으로 사용 가능하도록 설계했습니다.
  • 트리에 ReactNode를 직접 담으면 저장/복원이 불가능한 문제를 발견, componentKey와 ComponentStore를 도입해 JSON 직렬화만으로 레이아웃 저장/복원을 지원합니다.

드래그 앤 드롭 인터랙션 구현

  • 패널 중앙 / 부모 split 가장자리 / 루트 가장자리를 구분하는 depth 기반 드롭 타겟 판정을 구현했습니다.
  • 커서가 레이아웃 밖 이탈 후 재진입 시 드롭 preview가 복원되지 않는 문제를 preview 초기화·드래그 종료 로직 분리로 해결했습니다.

터치/펜 입력 지원

  • 마우스 전용 이벤트라 터치 기기에서 동작하지 않던 리사이즈·드래그를 Pointer Events 기반 단일 경로로 전환, setPointerCapture와 pointercancel 처리로 안정성을 확보했습니다.
  • 터치에서는 롱프레스로 드래그를 시작하고 elementFromPoint로 드롭 타겟을 판정하는 하이브리드 경로를 구현했습니다.
  • hover가 없는 터치 기기에서 리사이저가 보이지 않는 문제를 미디어쿼리(hover: none)로 해결했습니다.

Js-Te

JavaScript Testing Library (NPM Package)
JavaScriptBabel CompilerRollupNpm package
NPM PageDev Blog

Overview

Jest에서 영감을 받아 제작한 JavaScript 테스트 라이브러리입니다. 모킹 기능과 다양한 사용자 환경 지원에 집중했으며, Weekly Downloads 총합 3,300+를 달성했습니다.

모킹(Mocking) 기능 구현

  • 모듈 모킹에 전체 파일의 import문 수정이 필요했던 문제를 Babel plugin의 import 구문 조작으로 해결했습니다.
  • 동적 import 제약 해결을 위한 Wrapper 패턴을 적용했습니다.
  • Mock 경로 사전 수집을 통한 선별적 모듈 변환으로 성능을 최적화했습니다.

다양한 환경 지원

  • ESM만 지원하던 구조를 Rollup 기반 dual package 빌드로 ESM/CommonJS 동시 지원으로 확장했습니다.
  • 모듈 경로 작성 방식에 따라 모킹이 작동하지 않는 문제를 Babel 기반 절대 경로 정규화로 해결하고, 사용자 환경에 따른 자동 모듈 선택 로직을 구현했습니다.

테스트 프레임워크 핵심 기능 구현

  • test · describe · expect 등 Jest와 유사한 API, beforeEach·test.each 등 테스트 라이프사이클 훅을 구현했습니다.
  • CLI 도구를 통한 자동 테스트 파일 탐색 및 실행을 지원합니다.

JuGa (주식 모의 투자 서비스)

Real-time Stock Simulation Service (Team Project)
TypeScriptReactCanvas APISocket.ioShared Worker
GitHubOptimization Log

Overview

네이버 부스트캠프 최종 팀 프로젝트로 진행한 실시간 주식 모의 투자 서비스입니다. 외부 차트 라이브러리에 의존하지 않고 시각화와 실시간 데이터 처리를 직접 구현하는 데 집중했습니다.

Canvas 주식 차트 직접 구현

  • 외부 라이브러리 없이 Canvas API로 실시간 주식 차트를 직접 구현했습니다.
  • 대량의 시세 데이터 렌더링을 requestAnimationFrame 기반으로 처리해 부드러운 인터랙션을 유지했습니다.

경험의 확장 — 프로젝트 종료 후 웹소켓 연결 최적화

JuGa에서 여러 탭 사용 시 탭마다 웹소켓이 중복 연결되는 문제를 발견했지만 기간 내 해결이 어려웠고, 종료 후 타 프로젝트 리팩토링에 자발적으로 참여해 최적화를 완수했습니다.
  • 기존 소켓 아키텍처를 분석하고 Manager 계층을 추가해, 기존 코드를 유지한 채 Shared Worker 기반 구조로 점진적으로 전환했습니다.
  • Message 기반 Worker를 위한 이벤트 필터링을 추가하고, 연결 전 이벤트 처리 오류는 waitingQueue 도입으로 해결했습니다.
  • N개의 탭에서 발생하는 N개의 웹소켓 연결을 1개로 최적화했습니다.

한글 검색 & Composition 이벤트 이슈 해결

  • 한글 IME 입력 시 크롬·사파리 간 composition 이벤트 동작 차이로 발생하는 검색 중복 실행 문제를 디버깅하고 해결했습니다.
